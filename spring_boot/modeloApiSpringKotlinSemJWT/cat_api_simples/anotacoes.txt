2. Criar container Oracle Database:
docker pull container-registry.oracle.com/database/free:latest
docker run -d --name oracle-db -p 1521:1521 -p 5500:5500 -e ORACLE_PWD=SenhaForte123 container-registry.oracle.com/database/free:latest

Driver: Oracle
Connection Type: Service Name
Hostname: localhost
Port: 1521
Service: FREEPDB1
Driver: Thin
Authentication: User & Password
User: system
Password: SenhaForte123
Testar conexão, e confirmar

Criar usuário e senha para o projeto (executar 1 linha por vez):
CREATE USER catapi IDENTIFIED BY "CatApi@123";
GRANT CONNECT TO catapi;
GRANT RESOURCE TO catapi;
GRANT CREATE SESSION TO catapi;
GRANT CREATE TABLE TO catapi;
GRANT CREATE VIEW TO catapi;
GRANT CREATE SEQUENCE TO catapi;
GRANT CREATE TRIGGER TO catapi;
GRANT CREATE PROCEDURE TO catapi;
GRANT UNLIMITED TABLESPACE TO catapi;

Conectar com usuário criado (mesmas configurações acima, mas com usuário e senha criados):
Usuário: catapi
Senha: CatApi@123

Executar teste SQL: SELECT USER FROM DUAL;
5.1. Deverá retornar: CATAPI

32. Testar listagem de gatos (GET): http://localhost:8080/cats

33. No Postman, adicionar gato (POST): http://localhost:8080/cats
Body (raw JSON):
{
    "name": "Mingau",
    "breed": "Persa",
    "age": 3
}

34. Testar visualizar gato específico (GET): http://localhost:8080/cats/1

35. No Postman, editar gato (PUT): http://localhost:8080/cats
Body (raw JSON):
{
    "name": "Mingau ATUALIZADO",
    "breed": "Persa",
    "age": 3
}

36. No Postman, adotar gato específico (PATCH): http://localhost:8080/cats/1/adoption
Body (raw JSON):
{
    "adopterName": "Mateus"
}

37. No Postman, remover adoção de gato específico (DELETE): http://localhost:8080/cats/1/adoption
Body (raw JSON):
{
    "reason": "Adotante desistiu da adoção"
}

38. No Postman, remover gato específico (DELETE): http://localhost:8080/cats/1
